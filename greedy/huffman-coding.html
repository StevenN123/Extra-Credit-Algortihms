🔤 Huffman Coding

Greedy Algorithm #2 - Optimal Data Compression

📋 Problem Description

Given a set of characters and their frequencies, generate optimal prefix-free binary codes to minimize the total number of bits needed for encoding. Huffman coding is a lossless data compression algorithm used in formats like JPEG, MP3, and ZIP.

Formal Definition:

  • Given: Characters c₁, c₂, ..., cₙ with frequencies f₁, f₂, ..., fₙ
  • Find: Binary codes for each character (prefix-free) that minimize Σ fᵢ × length(codeᵢ)
  • Constraint: No code is a prefix of another code (prefix-free property)
O(n log n)
Time Complexity
O(n)
Space Complexity
Lossless
Compression Type

Key Properties

Prefix-Free No code is a prefix of another code
Optimal Minimizes weighted path length
Lossless Perfect reconstruction possible
Greedy Choice Always merge smallest frequencies

Algorithm Steps

  1. Count Frequencies: Count occurrence of each character
  2. Create Priority Queue: Create leaf nodes for each character
  3. Build Huffman Tree: Repeatedly merge two smallest nodes
  4. Generate Codes: Traverse tree (left=0, right=1)
  5. Encode: Replace each character with its binary code
  6. Decode: Traverse tree using bits to recover text
